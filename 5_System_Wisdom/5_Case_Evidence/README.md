# Case Evidence

**Purpose:** Documented episodes that illustrate CDPID principles in action. Each case is a bounded, narratively complete example that serves as both research evidence and teaching material.

**What Lives Here:**

- **Case Studies** — Detailed accounts of specific collaborative episodes with analysis of what CDPID dynamics were at play
- **Teachable Moments** — Shorter observations documenting a single principle demonstrated in practice
- **Publication Source Material** — Raw evidence base for white papers, articles, and curriculum development

**What Makes a Good Case:**

- A bounded episode with clear beginning, process, and outcome
- Observable CDPID dynamics (productive friction, Path A/B choice, emergent capability, failure mode and recovery, equilibration shift)
- Enough detail to be independently understandable without requiring the full session context
- Clear connection to one or more CDPID theoretical claims

**Existing Cases:**

- V4 Orchestration Architecture — Progressive intent discovery from "run session-startup" to complete architectural redesign (documented in CDPID Synthesis Document Part 9)
- CDPID Phase One Publication — Multi-platform research distribution demonstrating operational orchestration and system learning

**Folder Contents:**

- `Case_Index.md` — List of all documented cases with one-line descriptions and CDPID principles illustrated
- `YYYY-MM-DD_[Case-Name].md` — Individual case documents
- `_Archive/` — Cases incorporated into publications or superseded by more complete versions
