# 9_Daily_Debriefs

**Created:** 2026-02-22
**Purpose:** Consolidated daily work summaries — one file per day, covering all task threads completed, learnings captured, and System Wisdom gained.
**Status:** Active

---

## What Lives Here

One `YYYY-MM-DD_Daily_Work_Summary.md` file per working day. Each file captures:
- All task threads completed that day (from SuperContext Layer 5D checkpoint data)
- Learnings captured (from SuperContext Layer 4)
- System Wisdom gained (new patterns, milestones, optimization principles added to `4_System_Intelligence/`)

## How It's Generated

Invoke the `daily-debriefing` prompt at end of day. It reads the current SuperContext and `Current_Intelligence.md`, then compiles the daily record automatically.

## File Pattern

`YYYY-MM-DD_Daily_Work_Summary.md`

## Why It's in System Wisdom

Daily debriefs live here rather than `7_Retained_Working_Files/` because the primary value is longitudinal — seeing how work, learnings, and behavioral development accumulate over time. This folder is the daily layer of the System Wisdom accumulation stack:

- **Daily:** `9_Daily_Debriefs/` — what happened today
- **Weekly:** `8_Weekly_Reviews/` — synthesis of the week's patterns
- **Ongoing:** `4_System_Intelligence/` — validated patterns and milestones as they develop
