# System Intelligence

**Purpose:** Accumulated patterns, validated principles, and optimization insights extracted from the three behavioral taxonomies over time. This is the wisdom layer — not raw data (that lives in the taxonomy snapshots), not the reference map (that lives in the comprehensive taxonomies), but what the system has learned about how to optimize itself.

**What Lives Here:**

- **Validated Patterns** — Behavioral correlations confirmed across multiple snapshots (e.g., "interpretation substitution decreases when context richness exceeds Level 3")
- **Optimization Principles** — Tested rules for improving system performance (e.g., "retrospective dyadic scoring after 3+ exchange sequences produces more accurate assessment than after single exchanges")
- **Anti-Patterns** — Documented failure modes and the conditions that produce them, with prevention strategies
- **Developmental Milestones** — Observed stage transitions in any taxonomy, what triggered them, and what they looked like
- **Calibration Data** — How AI estimates compare to actual outcomes; how scoring accuracy improves over time

**How Intelligence Is Generated:** Periodically (weekly or at significant moments), the accumulated taxonomy snapshots are reviewed for cross-cutting patterns. The System Intelligence folder captures what those patterns mean — not the raw observations, but the conclusions drawn from them.

**Folder Contents:**

- `Current_Intelligence_SW_v1.md` — Active principles and patterns currently informing system operation
- `YYYY-MM-DD_Intelligence_Update.md` — Timestamped additions when new patterns are validated
- `_Archive/` — Older intelligence updates consolidated into Current_Intelligence
