# Level 5: Content Strategy

**What lives here:** The bridge between offering specifications and actual asset production. One subfolder per offering, mirroring Level 4's organization.
**Organization:** By offering name (Option A) — matching Level 4.
**Update frequency:** Asset inventory monthly; strategy quarterly.

---

## Folder structure (mirrors Level 4):

```
Level_5_Content_Strategy/
  Academy-Course-1/
    Content_Strategy.md
    Asset_Inventory.md
    Messaging_Templates.md
    Content_Calendar.md
  Coaching-6-Month-Program/
    Content_Strategy.md
    Asset_Inventory.md
  Community-Membership/
    Content_Strategy.md
    Asset_Inventory.md
```

---

## What each content strategy contains:

- **Content_Strategy.md** — Content pillars (3-5 core themes), customer journey content map (what's needed at each of the 5 stages), distribution strategy, creation guidelines, voice/tone for this offering
- **Asset_Inventory.md** — Complete list of every content asset needed: name, type, purpose (which journey stage), status (not started / in progress / complete / published), due date, location/URL
- **Messaging_Templates.md** — Standard ways to describe this offering: elevator pitch, one-paragraph, social bio, landing page headline, key benefits, objection responses, FAQ answers
- **Content_Calendar.md** — What gets created when, publication schedule, promotion schedule, evergreen vs. timely

---

## Asset status tracking:
Status tracked within Asset_Inventory.md as a file-level field — NOT via folder location.
Folder-based status queuing lives in Level 6 (production), not here.

## How this level connects:
- UP to Level 4: Informed by offering specifications
- DOWN to Level 6: Determines which assets to produce and in what order
- ACROSS Level 5: Assets may serve multiple offerings (noted in inventory)
