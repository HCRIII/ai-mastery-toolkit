## Overview

This folder contains the documents that define who the user is, what the AI needs to know, and the structured business context organized through the Context level hierarchy.

## Key Documents

- **Personal_Preferences_v26.md** — User identity, working style, rules of engagement
- **SuperContext_v26.md** — Central operational state document, updated every session
- **Level_0_Map_&_Preferences/_AI_Context_Index_CX_v6.md** — Index of documents loaded into AI context
- **Level_0_Map_&_Preferences/_Human_Context_Index_CX_v1.md** — Index of documents the human references

## Context Files

Levels 0–6 organize business context from architecture down to asset production:

- **Level_0_Map_&_Preferences/** — System-wide structural overview
- **Level_1_Foundation/** — Personal and business foundation documents
- **Level_2_Strategy/** — Strategic planning documents
- **Level_3_Portfolio/** — Portfolio-level documentation
- **Level_4_Offering_Briefs/** — Individual offering specifications
- **Level_5_Content_Strategy/** — Content strategy by channel
- **Level_6_Asset_Production/** — Production-ready assets and content

Each Context level has its own `_README.md` describing scope and contents.
