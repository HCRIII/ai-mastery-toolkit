# Doc Coauthoring Skill - Structured Documentation Workflow

## What This Skill Does

Guides users through a structured workflow for co-authoring documentation including proposals, technical specs, decision docs, and similar structured content with efficient context transfer, iteration, and reader verification.

## When to Use This Skill

**Automatic triggers:**
- "write documentation", "create proposal", "draft spec"
- Documentation, proposal, technical spec requests

**Use when you need:**
- Professional documentation
- Technical specifications
- Business proposals
- Decision documents
- Project documentation
- Structured written content

## How to Invoke

**Explicit invocation:**
```
Use the doc-coauthoring skill
```

**Trigger words (automatic):**
- "Write documentation for..."
- "Create a proposal for..."
- "Draft a technical spec for..."
- "Help me write a decision doc..."

## Capabilities

- **Structured Workflow:** Step-by-step process
- **Context Gathering:** Efficient information transfer
- **Iteration:** Refine through feedback
- **Reader Verification:** Ensure doc serves audience
- **Professional Formatting:** Appropriate structure
- **Content Development:** Organized, clear writing

## Example Usage

### Example 1: Technical Specification
**You say:** "Help me write a technical spec for our new API authentication system"

**What happens:**
- Skill guides through workflow
- Asks clarifying questions
- Structures specification sections
- Iterates based on feedback
- Verifies completeness
- Returns polished spec

### Example 2: Project Proposal
**You say:** "Create a proposal for implementing a new customer feedback system"

**What happens:**
- Skill gathers requirements
- Structures proposal sections
- Develops content iteratively
- Ensures reader needs met
- Returns professional proposal

### Example 3: Decision Document
**You say:** "Write a decision doc for choosing between cloud providers"

**What happens:**
- Skill structures decision framework
- Gathers comparison criteria
- Develops analysis
- Documents recommendation
- Returns decision doc

## Tips for Best Results

**Prepare:**
- Know your audience
- Have key information ready
- Understand document purpose
- Identify required sections

**During workflow:**
- Answer clarifying questions fully
- Provide examples when helpful
- Give feedback on iterations
- Verify against reader needs

## Integration

**Works well with:**
- **docx skill:** Export to Word format
- **pdf skill:** Create final PDF versions
- **Templates:** Use document templates

## Common Use Cases

- Technical specifications
- Architecture decision records
- Project proposals
- Business cases
- Process documentation
- Product requirement documents
- Design documents
- Strategic plans
