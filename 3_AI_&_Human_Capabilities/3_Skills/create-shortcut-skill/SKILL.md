# Create Shortcut Skill - Automation Shortcuts

## What This Skill Does

Creates or manages shortcuts that can be run on demand or automatically on a schedule for task automation and workflow efficiency.

## When to Use This Skill

**Automatic triggers:**
- "create shortcut", "automate task"

**Use when you need:**
- Automate repetitive tasks
- Schedule recurring actions
- Create on-demand shortcuts
- Build workflow automations
- Set up background tasks

## How to Invoke

**Explicit invocation:**
```
Use the create-shortcut skill
```

**Trigger words (automatic):**
- "Create a shortcut to..."
- "Automate this task..."
- "Set up a scheduled shortcut for..."

## Capabilities

- **On-Demand Shortcuts:** Run when triggered
- **Scheduled Shortcuts:** Automatic on schedule
- **Task Automation:** Automate repetitive work
- **Workflow Integration:** Connect multiple steps
- **Background Execution:** Run without manual trigger

## Example Usage

### Example 1: On-Demand Shortcut
**You say:** "Create a shortcut to generate my weekly status report"

**What happens:**
- Skill creates shortcut definition
- Configures report generation steps
- Sets up trigger mechanism
- Returns ready-to-use shortcut
- Documents how to run it

### Example 2: Scheduled Automation
**You say:** "Set up a shortcut to backup my project files every Friday at 5pm"

**What happens:**
- Skill creates scheduled shortcut
- Configures backup steps
- Sets Friday 5pm schedule
- Returns automation setup
- Explains how to modify schedule

### Example 3: Workflow Shortcut
**You say:** "Make a shortcut that: checks email, summarizes new messages, and adds action items to my todo list"

**What happens:**
- Skill creates multi-step shortcut
- Chains actions together
- Configures workflow
- Returns complete automation
- Documents how to trigger

## Tips for Best Results

**Specify:**
- Exact actions needed
- Trigger type (on-demand vs scheduled)
- Schedule if automated (day/time)
- Input requirements
- Expected outputs

**For best shortcuts:**
- Break complex tasks into clear steps
- Test with sample data first
- Document trigger method
- Plan for error handling
