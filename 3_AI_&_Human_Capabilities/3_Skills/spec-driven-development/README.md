# Spec-Driven Development (SDD) Skill

## Overview

The Spec-Driven Development workflow is a systematic methodology for evaluating, refining, and implementing projects through four structured phases that prevent drift and ensure strategic alignment.

## When to Use This Skill

Use SDD when you need to:
- Evaluate existing work against strategic intent
- Refine specifications before implementation
- Identify and resolve misalignments systematically
- Make strategic decisions with full context
- Prevent building the wrong thing

## Quick Start

**To invoke this skill in a conversation:**
```
I want to use the Spec-Driven Development workflow to evaluate [project/system/initiative]
```

**The AI will guide you through:**
1. **Phase 0:** Intent Expression - Document current vs. desired state
2. **Phase 1:** Research - Understand why gaps exist
3. **Phase 2:** Interviews - Clarify strategic decisions
4. **Phase 3:** Design - Create actionable recommendations

## What's Included

- `SKILL.md` - Complete workflow instructions for AI
- `README.md` - This file (user documentation)
- `templates/` - Template documents for each phase

## Core Principles

1. **Front-load discovery** - Spend time understanding before building
2. **Research root causes** - Don't just identify gaps, understand why
3. **Clarify decisions** - Present options with tradeoffs, not just recommendations
4. **Update upstream** - Keep strategic documents as source of truth
5. **Prioritize ruthlessly** - Quick wins vs. strategic pivots
6. **Validate assumptions** - Interview phase catches misunderstandings

## Expected Outcomes

After completing SDD, you'll have:
- ✅ Clear understanding of current state vs. strategic intent
- ✅ Root cause analysis of any misalignments
- ✅ Strategic decisions made with full context
- ✅ Prioritized action plan with sequencing
- ✅ Updated strategic documents reflecting evolved strategy
- ✅ Quick wins identified for immediate execution
- ✅ Long-term roadmap for strategic pivots

## Time Investment

Typical timeline:
- **Phase 0:** 30-60 minutes (Intent Expression)
- **Phase 1:** 1-2 hours (Research with parallel agents)
- **Phase 2:** 60-90 minutes (Interview Q&A)
- **Phase 3:** 1-2 hours (Design recommendations)

**Total:** 4-6 hours to prevent weeks/months of building the wrong thing.
