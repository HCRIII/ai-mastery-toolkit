## Purpose

Framework files are the conceptual guides for each operational cluster — what a system is, why it exists, when to engage it, and what degrades when it is absent. Every framework has a paired workflow in `9_Workflows/` that governs procedural execution. The same `##_` prefix connects paired files across both folders.

## Naming Convention

- All framework files: `##_TitleCase_Name_Framework.md`
- Prefix range `01–09`: core system frameworks (paired with `9_Workflows/` and `8_Specifications/` counterparts at matching prefix)
- Prefix range `90–99`: templates, examples, and reference material (sort to bottom; includes standalone frameworks without workflow pairs)
- The paired workflow file is named in the framework's **Procedural Counterpart** section; the framework file is named in the workflow's **Framework Document** section
