## Description

`content-vector` identifies the highest-leverage content initiative from your active projects, maps the complete asset set top-down from strategic intent to individual assets × channels, and produces all in-scope assets in a single session — using your loaded toolkit context as the multiplier.

This Operation applies the **Effort Inversion Principle**: once brand voice, audience context, key messages, and framework IP are loaded, the marginal cost of each additional content asset approaches zero. The session produces at scale because the hard work — context architecture — is already done.

---

## What It Does

1. **Step 0 — Load content context (silent):** Brand Guidelines, Key Message Platform, Ideal Client Profiles, active projects from SuperContext, and relevant offering/content strategy files
2. **Step 1 — Identify highest-leverage initiative:** Scans active projects for urgency, compound strategic impact, and context richness; presents the top initiative for user confirmation
3. **Step 2 — Build full asset map:** Maps all content types and individual assets × channels top-down from the initiative; flags concurrent thread opportunities
4. **Step 3 — Confirm session scope:** User reviews the map and confirms which assets to produce now, which to run in parallel threads, and which to defer
5. **Step 4 — Produce all in-scope assets:** Produces every confirmed asset as a file, applying loaded context throughout; saves to the appropriate Level 6 production folder
6. **Step 5 — Session summary:** Lists all files produced, deferred assets, thread assignments, and recommended next action

---

## When to Use

- When you're ready to produce content for community deployment, CDPID Phase Two, or any active content initiative
- When you want to produce a full suite of assets for a campaign rather than one piece at a time
- When context is loaded and the question is "what should we produce and how much can we get done right now?"
- As a regular cadence operation for content-driven priorities (e.g., "run content-vector" at the start of any content-focused session)

---

## What You'll See

- A one-sentence initiative identification with rationale — confirm or redirect
- A full top-down asset map organized by content type and channel — including concurrent thread suggestions for large batches
- A confirmed scope list before production begins
- Completed asset files saved to `4_Context_Files/Level_6_Asset_Production/` as production proceeds
- A session summary with all file names, locations, and recommended next action

---

## Output

All assets produced are saved as individual `.md` files in `4_Context_Files/Level_6_Asset_Production/[Offering-Name]/[Initiative-Name]/02_Draft/`. Final confirmed assets move to `04_Complete/` after review.
