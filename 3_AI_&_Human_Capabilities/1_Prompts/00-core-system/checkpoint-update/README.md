## Description

After significant milestones, this prompt reviews what was accomplished and updates all relevant system files in place. It does not generate a separate checkpoint file — it writes the learning directly into SuperContext, Current Intelligence, Current State files, and Mastery Records. Ends with a priority confirmation.

## What It Does

1. **Review Session** — Analyzes recent work for learnings, decisions, patterns, breakthroughs
2. **Gather State** — 3 parallel agents read SuperContext, Current Intelligence, and Behavioral State files
3. **Execute Updates** — Directly updates SuperContext (layers 4, 5D, 6), Current Intelligence (milestones, patterns, tracking list), Current States, and Mastery Records as warranted. Echoes all changes to user.
4. **Priority Confirmation** — Asks if priorities need adjustment based on the update

## When to Use

- After completing a major deliverable or system change
- After resolving a significant problem
- After a breakthrough insight or paradigm shift
- Before shifting to a fundamentally different work stream
- When you say "lock this in" or "checkpoint"
