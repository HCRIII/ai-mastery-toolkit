## Overview

This category contains 2 prompts focused on accelerating workflows and optimizing approval processes. These prompts help move faster through trusted patterns and reduce unnecessary back-and-forth.

---

## Available Prompts

### 01-compound-approval-with-options
**Purpose:** Combine multiple decision points into a single approval request

**When to Use:**
- Multiple sequential decisions needed
- Options can be presented upfront
- Want to reduce approval rounds
- User preferences known but choice needed

**Pattern:** Pre-structured Option Selection (Stage 3)

**Example Use Case:**
Instead of asking "Should I use approach A?" then "What about detail B?" then "And C?", present all options at once: "Here are 3 approaches with trade-offs. Which do you prefer?"

---

### 02-workflow-acceleration
**Purpose:** Speed up execution after trust is established

**When to Use:**
- Repeated similar tasks
- Established working relationship
- Patterns are proven (Stage 3)
- Time is constrained

**Pattern:** Workflow Acceleration After Trust (Stage 3)

**Example Use Case:**
After completing similar reports 5+ times, switch from "Here's my plan, approve?" to "I'm following our established pattern and will share the completed report."

## Usage Guidelines

### When to Use Compound Approval
 **Use when:**
- You have 2-4 options to present
- Trade-offs are understood
- User needs to make choice but all options are good
- Want to save time on back-and-forth

 **Don't use when:**
- Only one clear approach exists
- Decision is binary (yes/no)
- User needs extensive explanation first

---

### When to Use Workflow Acceleration
 **Use when:**
- Task completed successfully 5+ times
- Pattern is documented (Stage 3)
- User trusts your approach
- Efficiency is valued

 **Don't use when:**
- New type of work
- User wants visibility into process
- Significant deviations from pattern
- Stakes are high/errors costly
