## What This Prompt Does

The toolkit accelerator has one purpose: make the toolkit more powerful with every use. It does this through three independent modes:

**Mode 1 — SDD Compliance Scan**
Audits all active projects against Spec-Driven Development v2 phase requirements. Surfaces gaps. Produces a compliance table and targeted fix proposals.

**Mode 2 — Intake Gate**
Processes incoming material (conversations, notes, ideas, files) through a four-outcome metabolic decision tree before it enters the toolkit: Assimilate / Transform / Queue / Reject. Nothing gets in without a known destination and a clear purpose.

**Mode 3 — Acceleration Check**
Audits the toolkit's compounding mechanisms: Reuse Ratio, Feedback Loop Closure Rate, and Phase 6 Closure Rate. Produces an Acceleration Score and a single highest-leverage action.

Invoke any combination of modes based on what the session requires.

---

## Key Distinction

This prompt is **not** `toolkit-integrity-update`. That prompt checks structure (README/INDEX presence, naming conventions, stale references). This prompt checks **power** — are projects SDD-compliant, is incoming material getting assimilated correctly, and is completed work actually compounding into future capability?

Run `toolkit-integrity-update` for structure. Run `toolkit-accelerator` for power.

---

## When to Invoke

- Before adding any new material to the toolkit (Mode 2 — always)
- After completing a major project (Mode 1 + Mode 3)
- When the toolkit feels cluttered or misaligned (all three modes)
- Monthly acceleration review (Mode 3 alone)
- Trigger phrases: "run the accelerator", "intake gate", "SDD audit", "toolkit check"
