## Phase 1: Context Load and Domain Selection (Silent → then output)

**Load the research reference silently:**
Read `4_Context_Files/Level_1_Foundation/1_Business_Foundation/D3_Intellectual_Property/Elite_Coach_Development_Research_CX1_v1.md` to establish the four-domain framework in active context.

**Then ask [User]:**

> Which of the four development domains is today's focus?
>
> 1. **Flow State Facilitation** — triggering and deepening flow in clients; transient hypofrontality; challenge-skills balance; Kotler's Flow Cycle
> 2. **Longevity Coaching** — healthspan compression; evidence-tiered protocols; Medicine 3.0 framework; behavioral change implementation
> 3. **Deliberate Practice** — reflective supervision; competency refinement; expert-vs-novice knowledge structures; 10,000-hour vs. 50-hour accelerated path

>
> Or: describe a specific client situation or coaching challenge you want to work through.

Wait for response before proceeding.

---

## Phase 2: Targeted Skill Development Work

Based on [User]'s selection, conduct a structured development session using the relevant research domain as the knowledge base.

**For domain selections (1–4):** Run a deliberate practice sequence:
1. Identify one specific sub-skill within the domain to focus on
2. Present the current research standard for that sub-skill (from the reference file)
3. Apply it to a real or hypothetical client scenario [User] provides
4. Give specific, immediate feedback on [User]'s approach
5. Propose one refinement — small, testable, applicable in the next session

**For client situation / challenge:** Run a case-based development session:
1. Gather the relevant context (client profile, presenting situation, what [User] tried)
2. Identify which of the four domains is most relevant to this situation
3. Surface the applicable research framework
4. Analyze [User]'s approach against the framework — what aligns, what gaps exist
5. Propose one concrete adjustment for the next session

**Standard throughout:** Apply the deliberate practice principle — this is not conversation, it is structured practice with immediate feedback. Each exchange should refine a specific, isolable skill element.

---

## Phase 3: Reflection and Pattern Capture

After the development work, run a brief structured reflection:

1. **What was practiced today?** (Name the specific sub-skill, not just the domain)
2. **What shifted?** (What [User] now sees or will do differently)
3. **What's the refinement target for next session?** (One forward-looking commitment)

Ask [User] to confirm the reflection summary before logging it. If [User] adds to or corrects it, integrate exactly as stated.
