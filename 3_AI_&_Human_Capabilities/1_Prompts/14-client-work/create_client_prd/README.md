## Description
Creates professional, client-facing Product Requirements Documents that bridge business needs with technical implementation. Integrates SuperContext for accurate scoping and applies appropriate communication style for client relationships.

## When to Use
- Starting a new client project or feature
- Documenting requirements for client approval
- Creating scope documentation for contracts or SOWs
- Establishing shared understanding with client stakeholders
- Defining success criteria for client deliverables

## Key Features
- SuperContext integration for timeline and capacity alignment
- Client-appropriate communication style (less internal, more professional)
- Business value framing for technical requirements
- Clear scope boundaries to prevent scope creep
- Realistic timelines with appropriate buffers
- Risk identification and mitigation strategies

## Input Requirements
- Problem or opportunity description
- Client business context and goals
- Technical constraints or preferences
- Desired timeline and budget parameters
- Success criteria from client's perspective

## Output Format
Professional PRD (4-8 pages) with:
- Executive summary
- Problem statement and solution overview
- User personas and use cases
- Functional and technical requirements
- Timeline with phases and milestones
- Assumptions, dependencies, and risks
- Success criteria and acceptance testing

## Usage Notes
- Lead with business value before technical details
- Use client's terminology (not internal jargon)
- Be realistic about timelines (include buffer)
- Clearly document what client needs to provide
- Frame technical complexity in business terms
- Ready to send directly to client stakeholders
