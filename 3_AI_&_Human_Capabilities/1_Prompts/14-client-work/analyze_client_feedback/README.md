## Description
Analyzes client feedback systematically to extract actionable insights, manage scope boundaries, prioritize improvements, and maintain healthy client relationships through strategic response planning.

## When to Use
- After receiving significant client feedback on deliverables
- During project check-ins or phase reviews
- When scope creep concerns emerge
- To identify patterns across multiple feedback instances
- Before planning response strategy for client concerns

## Key Features
- SuperContext integration for scope and capacity context
- Systematic categorization (bugs, features, scope changes, gaps)
- Pattern identification across feedback instances
- Scope management and creep prevention
- Relationship health assessment
- Prioritized action planning with rationale

## Input Requirements
- Client feedback (emails, meeting notes, messages, calls)
- Original project scope and requirements
- Current project status and timeline
- Any relevant context about client expectations

## Output Format
Structured analysis (up to 1500 words) including:
- Feedback categorization and prioritization
- Pattern and theme analysis
- Relationship health indicators
- Prioritized action plan (immediate, short-term, medium-term)
- Scope management recommendations
- Response strategy for client communication

## Usage Notes
- Separate feedback within scope from scope expansion requests
- Identify stated needs vs. underlying actual needs
- Be honest about scope boundaries and trade-offs
- Provide clear rationale for prioritization decisions
- Include both client-facing and internal recommendations
- Address relationship health proactively
