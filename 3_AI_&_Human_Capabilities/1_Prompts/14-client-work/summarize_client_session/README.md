## Description
Creates clear, actionable summaries of client sessions (calls, meetings, workshops) that document decisions, clarify next steps, prevent misunderstandings, and ensure all stakeholders maintain shared understanding.

## When to Use
- After every client call or meeting
- Following discovery or requirements sessions
- After project kickoffs or reviews
- Post-workshop synthesis
- When multiple stakeholders need alignment

## Key Features
- SuperContext integration for project context
- Clear decision documentation
- Specific action items (who, what, when)
- Scope/timeline impact identification
- Clarifications and open questions surfaced
- Professional, ready-to-send format
- Request for confirmation to ensure alignment

## Input Requirements
- Session type and attendees
- Topics discussed and outcomes
- Decisions made during session
- Action items identified
- Questions or clarifications needed
- Any scope or timeline impacts

## Output Format
Professional session summary (typically under 500 words) with:
- Session details and attendees
- Key outcomes (2-3 sentence summary)
- Topics discussed with key points
- Decisions documented
- Action items (client, us, shared) with owners and dates
- Scope/timeline impacts if any
- Clarifications needed
- Open questions for future discussion
- Next session planning

## Usage Notes
- Send within 24 hours while fresh
- Be specific about action items (avoid vague "follow up")
- Document decisions to prevent later confusion
- Surface ambiguous points that need clarification
- Include professional request for confirmation
- Ready to send to both client and internal team
- Use as accountability tool for commitments
- Helps prevent scope creep through clear documentation
