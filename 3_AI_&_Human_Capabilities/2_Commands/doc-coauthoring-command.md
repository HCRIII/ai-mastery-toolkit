# /doc-coauthoring Command

**Created:** 2026-02-13
**Last Updated:** 2026-02-13
**Purpose:** Command reference for Doc Coauthoring capability
**Status:** Final

---

**Command:** `Use the doc-coauthoring skill`

**What it does:** Guides users through a structured workflow for co-authoring documentation including proposals, technical specs, decision docs, and similar structured content with efficient context transfer, iteration, and reader verification.

---

## How to Invoke

**Explicit command:**
```
Use the doc-coauthoring skill
```

**Automatic triggers:**
- "writing docs"
- "creating proposals"
- "drafting specs"
- Similar documentation tasks

---

## When to Use

Use this command when user wants to write:
- Documentation
- Proposals
- Technical specs
- Decision docs
- Similar structured content

**This workflow helps:**
- Efficiently transfer context
- Refine content through iteration
- Verify the doc works for readers

---

## Example Invocations

**Example 1: Explicit command**
```
Use the doc-coauthoring skill to create a technical specification
```

**Example 2: Automatic trigger (documentation)**
```
Help me write documentation for our API
```

**Example 3: Automatic trigger (proposal)**
```
Create a proposal for the new customer portal
```

**Example 4: Automatic trigger (spec)**
```
Draft a technical spec for the authentication system
```
