## Overview

This folder contains the practical implementation layer of the AI Mastery Toolkit. Here you'll find the actual capabilities you can use, the commands that invoke them, the skills that combine them, and the frameworks that guide their application.

**Think of this as your capability library** - the tools, techniques, and patterns you can apply immediately in your AI collaboration work.

---

## Folder Organization

The 13 capability folders are organized by type and complexity:

### 0_Documentation_Standards
**The meta-capability.** Standards for creating AI-optimized documentation that makes all other capabilities more effective. Start here to understand how to structure knowledge for AI consumption.

### 1_Prompts
**Foundation capability.** Library of proven prompt patterns, templates, and techniques organized by use case. The building blocks of all AI interaction.

### 2_Commands
**Invocation layer.** Built-in slash commands and custom shortcuts for invoking skills and capabilities. Learn command syntax and create your own shortcuts.

### 3_Skills
**Pre-built systems.** Curated skills that combine prompts, context, and workflows into ready-to-use capabilities. Skills are like specialized AI assistants.

### 4_Projects_&_RAG
**Context management.** Patterns and techniques for managing knowledge, maintaining context across sessions, and building retrieval-augmented generation systems.

### 5_Agents
**Autonomous systems.** Multi-step workflows where AI operates independently to achieve goals. Agents handle complex tasks with minimal human intervention.

### 6_Connectors_MCP
**External integrations.** Model Context Protocol (MCP) servers that connect AI to external tools, APIs, and data sources.

### 7_Frameworks
**Mental models.** Structured thinking frameworks like SWOT, 5 Whys, First Principles, and decision-making patterns that guide AI collaboration.

### 8_Specifications
The precision layer between Frameworks and Workflows. Specification documents define exactly what a system, workflow, or deliverable must do — scoped, constrained, and expressed in testable requirements — before any workflow is designed. The 6-phase lifecycle (Define → Research → Specify → Review → Verify & Validate → Baseline) is the standard process for every specification in the toolkit.

### 9_Workflows
**Process patterns.** Multi-step sequences for common tasks like content creation, analysis, research, and development.

### 10_Automations
**Triggered actions.** Event-driven and scheduled automations that execute workflows automatically based on conditions or timing.

### 11_Orchestration
**System coordination.** Patterns for coordinating multiple agents, workflows, and capabilities to handle complex, multi-faceted challenges.

### 12_Artifacts
**Output formats.** Techniques for creating rich outputs like HTML, React components, diagrams, visualizations, and interactive content.

## How to Use This Folder

**If you're learning:** Start with 1_Prompts to understand the foundation, then progress through 2_Commands and 3_Skills to build practical capability.

**If you're building:** Navigate to the specific capability type you need (Frameworks for thinking patterns, Workflows for process templates, etc.).

**If you're integrating:** Explore 5_Agents, 6_Connectors_MCP, and 11_Orchestration for advanced system integration patterns.

**If you're creating content:** Reference 12_Artifacts for output format techniques and 0_Documentation_Standards for structuring knowledge.
